## 2026-06-13 — 1차 앱 구현 (`kuji-app/`)

PRD v1.1 기반으로 단일 HTML + PWA 앱을 구현하고, 실기 테스트 피드백을 받아 v1.2까지 반영.

### 1. 산출물 (`kuji-app/`)
- `index.html` — 앱 본체(화면·추첨·연출·관리자 폼 전부 인라인), `manifest.webmanifest`, `sw.js`, `assets/icon-192·512.png`, `README.md`(비개발자용 배포·운영 가이드)
- 데이터: 설정/재고/로그 = localStorage, 업로드 이미지·대기영상·BGM·뽑기NPC = IndexedDB

### 2. 구현 범위 (PRD v1.1 P0)
- 멀티 IP(최대 3개, 관리자 지정·IP별 독립 재고), IP 1개면 IP선택(S2) 생략
- S1~S6 플로우 + 모든 화면 뒤로가기 + 30초 무입력 자동복귀(S2·S3·S6)
- 비복원·100% 당첨 추첨 + **추첨 원자 락**(연타 방지)
- 관리자: 우상단 5탭+PIN, IP/등급/상품/수량 편집, 이미지 업로드, 재고 현황·리셋, 당첨 로그·CSV, 음소거, 미디어 업로드
- 스페셜 이펙트(금빛 번개+사운드)는 A/B/C/행운상에서만, 사운드는 WebAudio 합성(SFX)+업로드 BGM
- PWA 오프라인 캐싱(배포 도메인), **localhost에선 SW 자동 해제**(개발 중 캐시 문제 방지)

### 3. 실기 테스트 피드백 반영 (→ PRD v1.2)
| 피드백 | 반영 |

| 관리자 진입 안 됨 | CSS 우선순위 충돌(`#scr-idle` 그룹 규칙이 admin-tap 덮어씀) 수정 + 우상단 이동·영역 확대 |
| 옛 화면이 계속 보임 | 서비스워커 캐시 문제 → localhost SW 자동 해제 로직 추가 |

| 뽑기 화면 | 티켓 그리드 → **단일 "뽑기!" 버튼 + NPC 아트 박스**(이미지 미크롭) |

## 2026-06-14 — 오디오·NPC 보강 + 디자인 시스템 적용 플랜 (PRD v1.3)

### 1. 오디오 (BGM·효과음 관리자 지정)
- **BGM 화면별 3슬롯**: 대기 / 선택(IP·라인업) / 뽑기·연출·결과 — 화면 전환 시 자동 전환·루핑
- **효과음 업로드 3종**: 뽑기음 / 일반당첨 / 스페셜. 업로드 시 그 파일을, 없으면 **기본 합성음(WebAudio) 폴백** (= 디폴트는 이전과 동일)
- 음소거 토글은 BGM·효과음 모두 적용

### 2. 미디어 저장 효율화 (Blob)
- 대기영상·BGM·효과음·NPC를 **Blob 원본**으로 IndexedDB 저장(기존 base64 → 용량 1.3배 부풀던 문제 해결)
- 용량 상향: 대기영상 30→**150MB**, 오디오 8→**30MB**, NPC 영상 **60MB**
- 원인 추적: "어떤 BGM은 등록 안 됨" = 8MB 제한 + base64 인플레이션이었음 → 해결

### 3. 뽑기 NPC (랜덤·영상)
- NPC를 **여러 개 등록**(이미지/영상 혼용) → 뽑기 화면마다 **랜덤 표시**
- **직전에 나온 NPC는 연속 제외**(2개 이상일 때)
- 영상 NPC는 **무음 루핑** 자동재생
- 기존 단일 NPC(`draw_npc`)는 부팅 시 새 목록으로 **자동 이전**
- S4 NPC 표시 영역 **1.5배 확대**(285px / 태블릿 420px)

### 4. 디자인 시스템 적용 플랜 수립
- com2ustore.com 분석: 색·폰트가 CSS 변수로 노출 안 됨(이미지 위주 커머스 테마) → 자동 추출 한계 확인
- 합의: **내일 피그마 시안 → Claude 디자인 시스템 → MD 정제 → 적용**. 토큰 레이어 구축+적용은 MD 도착 후 한 번에
- 플랜 문서: `~/.claude/plans/toasty-growing-donut.md` (토큰 인벤토리, MD 스펙, 적용 단계, 검증)

### 5. 검증
- 매 변경마다 JS 문법 체크 통과, 로컬 서버 클릭 테스트(아이패드 프로 비율). 추첨 로직 무변경
