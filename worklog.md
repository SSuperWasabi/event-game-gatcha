## 2026-06-14 — 오디오·NPC 보강 + 디자인 시스템 적용 플랜 (PRD v1.3)

### 1. 오디오 (BGM·효과음 관리자 지정)
- **BGM 화면별 3슬롯**: 대기 / 선택(IP·라인업) / 뽑기·연출·결과 — 화면 전환 시 자동 전환·루핑
- **효과음 업로드 3종**: 뽑기음 / 일반당첨 / 스페셜. 업로드 시 그 파일을, 없으면 **기본 합성음(WebAudio) 폴백** (= 디폴트는 이전과 동일)
- 음소거 토글은 BGM·효과음 모두 적용

### 2. 미디어 저장 효율화 (Blob)
- 대기영상·BGM·효과음·NPC를 **Blob 원본**으로 IndexedDB 저장(기존 base64 → 용량 1.3배 부풀던 문제 해결)
- 용량 상향: 대기영상 30→**150MB**, 오디오 8→**30MB**, NPC 영상 **60MB**
- 원인 추적: "어떤 BGM은 등록 안 됨" = 8MB 제한 + base64 인플레이션이었음 → 해결

### 3. 뽑기 NPC (랜덤·영상)
- NPC를 **여러 개 등록**(이미지/영상 혼용) → 뽑기 화면마다 **랜덤 표시**
- **직전에 나온 NPC는 연속 제외**(2개 이상일 때)
- 영상 NPC는 **무음 루핑** 자동재생
- 기존 단일 NPC(`draw_npc`)는 부팅 시 새 목록으로 **자동 이전**
- S4 NPC 표시 영역 **1.5배 확대**(285px / 태블릿 420px)

### 4. 디자인 시스템 적용 플랜 수립

- 합의: **내일 피그마 시안 → Claude 디자인 시스템 → MD 정제 → 적용**. 토큰 레이어 구축+적용은 MD 도착 후 한 번에

## 2026-06-15 — iPad 테스트 환경 검토 + 디자인 시스템 전면 적용 (퍼플+골드)

### 1. iPad 테스트 환경 검토 & 아키텍처 재확인
- 로컬 서버(`python http.server`)로 PC 미리보기. iPad 실기 접속 시도하며 환경 한계 확인:
  - 이 PC는 **유선 전용(와이파이 어댑터 없음)** + 사내망 **기기 격리** → LAN 직접 접속 불가. 폰 핫스팟도 PC가 못 붙어 무력화.
  - 해결: **cloudflared quick tunnel**(winget 설치)로 임시 HTTPS 노출 → iPad 접속 성공(테스트 후 종료). 실제 행사 최종본은 GitHub Pages/Vercel 고정 배포 권장.
- **"랩탑 컨트롤타워 → 아이패드 실시간 반영"** 아이디어 검토 → **기각**. 현 시점 **아이패드 1대 가정, 오프라인-퍼스트 독립형 원안 유지**(앱이 이미 이 설계). 다중 태블릿(5~10대) 확장 시 **설정 백업(내보내기/가져오기) + 컨트롤타워**를 로드맵으로 보류.
