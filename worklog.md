### 1. 레퍼런스 영상 분석
- 입력 자료: `Reference/IMG_6182.MOV` — 이치방쿠지(一番くじ) 디지털 키오스크 실기 촬영 영상 (4K 120fps, 약 25초, 306MB)
- ffmpeg로 1초당 1프레임씩 캡처(`Reference/frames/`, 25장)하여 화면 플로우 분석
- 파악한 플로우:
  1. 대기 화면 — "화면을 터치하면 시작합니다"
  2. 상품 시리즈 목록 — IP별 가격·잔여량 표시 (예: 29/80)
  3. 등급별 경품 상세 — A상~J상, 등급별 잔여 수량, LAST ONE상, DOUBLE CHANCE상
  4. 참여 횟수 선택 — +/− 버튼, 합계 금액
  5. 카드 결제
- 핵심 메커니즘 식별: **비복원 추첨(쿠지 방식)** — 정해진 티켓 풀에서 뽑힌 티켓은 소멸, 마지막 티켓 당첨자에게 LAST ONE상 추가 증정

### 2. 시연용 프로토타입 제작
- 산출물: `kuji-event-game.html` (단일 HTML 파일, 브라우저에서 바로 실행)
- 구현 내용:
  - 레퍼런스 플로우 재현 (대기 → 시리즈 선택 → 경품 상세 → 횟수 선택 → 모의 결제) + 영상에 없던 티켓 뽑기 연출·결과 화면 추가
  - 비복원 추첨 로직, LAST ONE상, SOLD OUT 처리
  - localStorage 재고 영속화 (새로고침에도 유지), R 키 재고 리셋

### 3. 프로젝트 방향 확정
- 목적: **사내 팝업 스토어 임직원 대상 이벤트 게임 1차 프로토타입**
- 작업 분담: **Fable이 PRD 작성 → Opus가 PRD 기반으로 코딩** (크레딧 절약 전략)
- 논의로 확정한 운영 조건:
  | 항목 | 결정 |
  |---|---|
  | 운영 기기 | 태블릿 키오스크 1대 (iPad 또는 갤럭시탭, 세로 거치) |
  | 배포 형태 | 웹앱(PWA) — 홈 화면 설치, 전체화면 실행 |
  | 참여 방식 | 1인 1회, 시스템 인증 없음 (스태프 현장 통제) |
  | 결제 | 없음 (사내 무료 이벤트 — 레퍼런스의 결제 단계 제거) |
  | 경품 | 실물 굿즈, 수량 미확정 → config로 쉽게 수정 가능하게 |
  | 관리 기능 | 간단한 관리자 화면 (재고 현황·리셋·당첨 로그 CSV) |
