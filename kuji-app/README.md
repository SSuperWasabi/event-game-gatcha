# 컴투스 팝업 쿠지 — 운영 가이드

사내 팝업 스토어용 쿠지(복권식 가챠) 이벤트 게임. **단일 HTML + PWA**로, 태블릿 1대에서 서버 없이 오프라인으로 동작합니다.

> 비개발자도 운영할 수 있게 작성했습니다. 모든 콘텐츠(IP·상품·수량·이미지)는 **앱 안의 관리자 화면**에서 바꿉니다. 코드/파일을 직접 고칠 필요가 없습니다.

---

## 1. 파일 구성

```
kuji-app/
├── index.html            ← 앱 본체 (이 파일이 게임 전체)
├── manifest.webmanifest  ← PWA 설정 (앱 이름·아이콘·전체화면)
├── sw.js                 ← 오프라인 캐싱 (인터넷 없어도 동작)
└── assets/
    ├── icon-192.png       ← 홈 화면 아이콘
    ├── icon-512.png
    ├── bgm/               ← (선택) 기본 BGM 파일 둘 자리
    └── sfx/               ← (선택) 효과음 파일 둘 자리
```

업로드한 경품 이미지·대기 영상·BGM은 파일이 아니라 **태블릿 브라우저 안(IndexedDB)에 저장**됩니다. 관리자 화면에서 올립니다.

---

## 2. 배포 (처음 한 번)

PWA는 **HTTPS(보안 연결)**에서만 오프라인 기능이 켜집니다. 아래 둘 중 하나로 올리세요(둘 다 무료).

### 방법 A — GitHub Pages
1. 이 `kuji-app/` 폴더를 GitHub 저장소에 올립니다.
2. 저장소 Settings → Pages → Branch를 `master`(또는 main)로 지정 → 저장.
3. 잠시 후 생기는 주소(`https://<계정>.github.io/<저장소>/kuji-app/`)가 게임 URL입니다.

### 방법 B — Vercel
1. https://vercel.com 에 GitHub로 로그인 → New Project → 저장소 선택.
2. 배포 후 나오는 주소 뒤에 `/kuji-app/` 를 붙이면 게임 URL입니다.

> **로컬 테스트만** 할 때는 HTTPS가 없어도 됩니다(아래 5번 참고). 단, 오프라인/홈화면 설치는 HTTPS에서만 확인됩니다.

---

## 3. 태블릿 설치 (키오스크 만들기)

1. 태블릿(아이패드/갤럭시탭) 브라우저로 위 URL 접속 → 화면이 다 뜰 때까지 기다립니다(자산 캐싱).
2. **홈 화면에 추가**
   - 아이패드(Safari): 공유 버튼 → "홈 화면에 추가"
   - 갤럭시탭(Chrome): 메뉴(⋮) → "홈 화면에 추가" / "앱 설치"
3. 홈 화면에 생긴 **쿠지 아이콘**으로 실행하면 주소창 없이 전체화면으로 뜹니다.
4. (한 번 정상 실행하면) 이후 와이파이가 끊겨도 동작합니다.

### 키오스크 잠금 (참여자가 다른 앱 못 열게)
- **아이패드**: 설정 → 손쉬운 사용 → **가이드 접근(Guided Access)** 켜기 → 게임 실행 후 홈버튼/측면버튼 3번 클릭.
- **갤럭시탭**: 설정 → 보안 → **앱 고정(화면 고정)** 켜기 → 게임을 고정.

## 4. 운영 방법

### 관리자 화면 들어가기
- 대기 화면(첫 화면) **좌측 상단 모서리를 3초 안에 5번** 연속 터치 → PIN 입력 (기본 `0000`).

### 행사 전 준비 (관리자 화면)
1. **IP·상품 탭**: IP(최대 3개) 이름, 등급(A상~), 경품명, **수량**, 이미지를 입력/업로드합니다.
   - "상위"로 표시한 등급(보통 A·B·C상)이 당첨되면 금빛 번개 + 사운드 연출이 나옵니다.
   - **행운상**: 각 등급의 *마지막 1개*를 뽑은 참여자에게 스토어 할인 쿠폰(지류)을 **현장에서 지급**합니다. 행운상 이름/이미지는 "행운상" 칸에서 설정합니다.
   - 수량 합계가 곧 그 IP의 전체 티켓 수입니다. (모든 티켓 = 당첨, 빈 티켓 없음)
2. 입력 후 **💾 저장**을 누릅니다. 수량을 바꿨다면 "재고 초기화"에 **예**를 누르세요(정확한 재고를 위해).
3. **설정 탭**: 이벤트명, PIN, 유휴 시간(기본 30초), 대기 영상·BGM·**뽑기 NPC 이미지**를 올립니다.
4. **재고 탭**에서 수량이 맞는지 확인하고 운영을 시작합니다.

### 행사 중
- 참여자가 대기 화면을 터치 → IP 선택 → 경품 확인 → 뽑기 → 결과 화면을 **스태프가 옆에서 확인**하고 굿즈 지급 → "처음으로".
- **1인 1회는 스태프가 직접 통제**합니다(시스템은 한 번의 터치로 한 번만 뽑히도록 잠급니다).
- 30초간 아무 입력이 없으면 자동으로 대기 화면으로 돌아갑니다.

### 행사 후 (중요)
- **당첨로그 탭 → 📄 CSV 내보내기**로 당첨 기록을 백업하세요. (태블릿 분실/초기화 대비)
- 다음 날/다음 행사 전 필요 시 **재고 리셋**.

## 5. (개발자용) 로컬에서 빠르게 열어보기

서버 없이 `index.html`을 더블클릭해도 **기본 플로우는 동작**하지만, PWA/일부 기능은 로컬 서버에서 확인하는 게 정확합니다.

```bash
# Python이 있으면
cd kuji-app
python -m http.server 8000
# 브라우저에서 http://localhost:8000
```

수정 후 재배포 시에는 `sw.js`의 `CACHE = 'kuji-v1'` 버전을 올려야 태블릿이 새 버전을 받습니다.

## 6. 자주 묻는 것

| 질문 | 답 |
|---|---|
| 인터넷이 끊기면? | 처음 한 번 정상 로딩했다면 오프라인에서도 동작합니다. |
| 태블릿을 바꾸면 설정도 옮겨지나요? | 아니요. 설정·이미지는 그 태블릿에 저장됩니다. 새 기기에선 다시 입력해야 합니다. |
| 당첨 기록은 어디 있나요? | 그 태블릿 안에 저장됩니다. 행사 후 꼭 CSV로 백업하세요. |
| 소리가 안 나요 | 관리자 → 설정 → 음소거 확인. 또 화면을 한 번 터치해야 소리가 시작됩니다(브라우저 정책). |
| IP를 1개만 두면? | IP 선택 화면을 건너뛰고 바로 경품 라인업으로 갑니다. |
