# 상품 노출/숨김 토글 — 설계 (2026-07-07)

운영 요구: "랜덤 경품" 같은 긴급용 상품을 미리 세팅해두고, 필요할 때만 리스트에 노출/숨김 전환.

## 방식

- `prize.hidden` 플래그 + 상품 탭 행마다 **노출/숨김 토글 버튼** (번들 버튼 옆).
- **즉시 저장·즉시 반영** (순서변경 ▲▼과 동일) — 저장 버튼·재고 초기화 확인 없이 긴급 전환.
- 숨김 행은 관리자에서 흐리게(opacity .5) 표시.

## 숨김 상태의 동작

| 영역 | 동작 |
|---|---|
| S3 라인업 | 카드 미표시 (품절 아님, 아예 없음) |
| 추첨 풀 | 제외 — **쿨다운 폴백에서도 제외** |
| S2·전체 잔여 집계(remainOf/totalOf) | 제외 (SOLD OUT 판정도 노출분 기준) |
| 관리자 재고 탭 | (숨김) 라벨로 계속 표시 |
| 설정·재고·이미지 | 보존 — 노출 전환 시 그대로 복귀 |

- 대안 "잔여 0으로 두기"는 S3에 품절 카드로 노출되므로 부적합(참여자 혼란).
- 안전 가드: 노출 상품이 전부 소진된 상태에서 추첨 시도 시(숨김 전환 경합 등) 빈 풀이면 토스트 후 라인업 복귀.
- 행운상: 숨김 등급은 안 뽑히므로 마지막 1개 판정과 충돌 없음.

## 검증

- 시뮬: 노출분만 정확 소진, 숨김 재고 보존, 폴백 제외, 빈 풀 안전, 노출 전환 즉시 복귀.
- 헤드리스 스모크: 라인업 카드 수 감소/복귀, 잔여 집계, 15회 추첨 중 숨김 0회, (숨김) 라벨.
- `APP_VER v22`, `sw.js kuji-v22`.
