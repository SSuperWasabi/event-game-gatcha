# 번들 상품(Bundle Prizes) 설계

작성일: 2026-06-26 · 대상: `kuji-app/index.html` (단일 PWA)

## 목적
하나의 상품(등급) 안에 여러 하위 상품을 묶어 등록하고, 당첨 시 하위 중 1개를 무작위 지급하는 "번들 상품" 기능. 예: C상 "가디언스클럽 랜덤" → 가방/인형/목도리/후디 중 1개.

## 핵심 결정 (확정)
- **당첨 동작**: 번들 당첨 → 하위상품 중 **재고 비례 무작위 1개** 지급. 당첨화면엔 그 1개 썸네일/이름만 표시.
- **재고**: 하위상품별 **개별 추적**. 하위 재고 0이면 추첨 풀 자동 제외.
- **S3 라인업 표현**: 번들 **대표 이미지 1장** + 잔여(하위 합계)/초기 합계. 하위는 비공개(서프라이즈).

## 데이터 모델
- `prize`에 선택 필드 추가:
  - `bundle: true`
  - `subs: [{ name, imageKey, count }]`
  - 번들일 때 `prize.name` = 번들명, `prize.imageKey` = 대표이미지, `prize.count` = 하위 합계(파생), `prize.grade`/`tier` 유지.
- `stock[ipId][i]`: 단순상품이면 **숫자**, 번들이면 **배열**`[c0,c1,...]`(subs와 평행).
  - `freshStock`: `p.bundle ? p.subs.map(s=>+s.count||0) : (+p.count||0)`
  - `reconcileStock`: 모양 불일치(번들↔배열 길이) 보정
  - `remainOf`/`totalOf`: 배열은 flatten 후 합산

## 추첨 로직 (`pickDraw`)
- 풀 엔트리 = `{i, sub}` (단순=sub null, 번들=하위 index)
  - 단순상품 i: `{i,sub:null}`를 stock 수만큼 push
  - 번들상품 i: 각 하위 j를 `stock[i][j]`만큼 `{i,sub:j}` push (재고 비례 = 실물 1개 동일확률)
- 추첨 → 해당 칸 차감 (단순 `arr[i]--`, 번들 `arr[i][sub]--`)
- 표시용 resolved = 번들이면 하위 name/imageKey, 아니면 prize의 것
- 행운상(isLucky): 그 등급 잔여 합계가 0이 되는 순간 (번들은 하위 합계)
- `lastResult`에 resolved(이름/이미지) 포함 → S5/S6는 resolved로 출력, 프레임색/등급은 prize 기준

## 백오피스 (관리자 모드)
- 상품 행에 **`번들` 토글** 추가
- ON → 하위상품 편집 영역 펼침: 하위행마다 `경품명·이미지·조정·수량·삭제` + `+ 하위상품 추가`
- 상단 수량칸은 하위 합계 표시(직접입력 비활성)
- 번들 이미지/조정 = 대표이미지
- 신규 함수: `admToggleBundle, admAddSub, admDelSub, admEditSub, admPickSubImg`
- 저장/재고초기화 흐름 기존과 동일

## 화면 반영
- **S3 라인업**: 번들 대표이미지 + 잔여(합계)/초기(합계); 등급별 잔여 계산이 배열 stock 처리
- **S5/S6**: resolved 하위상품 이미지·이름 출력
- **재고 현황 탭**: 번들은 하위별 잔여 펼쳐 표시
- **CSV 로그**: `prizeName` = 실제 받은 하위상품명 (예: "가디언스클럽 랜덤 - 가방")

## 안전장치 / 호환
- 기존 단순상품 설정은 그대로 호환(무영향)
- 하위 전부 소진 → 등급 품절 표시 + 추첨 제외
