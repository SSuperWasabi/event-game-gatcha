# 라인업(S3) 표시 토글 + 뽑기 화면 자동복귀 — 설계 (2026-07-07)

팀 운영 피드백: 운영 효율 극대화를 위해 S3 경품 라인업을 통째로 숨기고 싶다.

## 방식 결정 — 저장소 카피 대신 본편 토글

- 검토안: (A) 깃허브 카피 + 축약 버전 별도 운영 / (B) 본편에 화면 표시 토글.
- **B 채택.** 카피는 이중 유지보수(하루 3회 배포 페이스), PWA 2개 혼선, `.kuji` 호환 분기, Pages 배포 세팅 중복 → 기각.
- 모델: **S2 = 자동**(기존 — IP 1개면 생략), **S3 = 수동 토글**(운영자 선택).

## 기능 1 — 라인업(S3) 표시 토글

- `cfg.showLineup` (기본 true). 관리자 설정 탭 토글 버튼 — **즉시 저장·즉시 반영**.
- 숨김 시 플로우: S1 터치 → (IP 여러 개면 S2) → **S4 뽑기 직행** (`enterGame` 분기).
- S4 뒤로가기(`drawBack`): 표시 중 → S3 / 숨김 → S2(IP 1개면 S1).
- 참고: 숨김 시 참여자는 경품 목록·행운상 안내를 화면에서 못 봄 — 현장 인쇄물/스태프 안내 전제.

## 기능 2 — S4 뽑기 전 자동복귀

- 배경: S3 숨김 시 S4가 대기 지점이 되는데 기존 S4는 무입력 복귀가 없어 이탈 시 영구 대기.
- `cfg.drawIdleSec` (기본 **5초**, 0=끔, 관리자 설정) — 현장 운용 속도가 빨라 짧게(Jason). 조절 가능한 다이얼로.
- **뽑기 버튼 누르기 전 상태에만** 적용: 타이머 콜백이 `drawing` 플래그 확인 + `pickDraw` 진입 시 타이머 해제 → 추첨 확정 후 연출·결과 중 복귀 절대 없음.
- S3 표시 여부와 무관하게 항상 적용(동작 일관성).

## 검증

- 헤드리스 스모크: 기본 S1→S3 / 숨김 S1→S4 직행 / 뒤로가기 S1 / 2초 설정 자동복귀 / 뽑기 중 복귀 없음(S6 정상 도달) / 토글 복원. 예외 0.
- `APP_VER v23`, `sw.js kuji-v23`.
