# 잔여 수량 실시간 조정 + 상위 등급 쿨다운 — 설계 (2026-07-07)

운영 피드백 2건 반영. 대상: `kuji-app/index.html` (+ `sw.js` 캐시 버전).

## 배경 (운영 중 발견)

1. 상품 탭의 '수량' 칸은 **초기 수량**이라 뽑혀도 줄지 않음 → 운영자가 잔여를 실시간으로 파악·조정할 수 없음.
2. 상위 등급(A/B/C/스페셜)이 단시간에 연달아 당첨되는 것을 막을 장치가 없음.

## 기능 1 — 상품 탭 잔여 수량 표시·즉시 조정

- 상품 행: `초기 [n]` 칸 옆에 **`잔여 [n]`** 칸 추가 (골드 테두리로 구분).
  - 잔여 칸 = `stock` 실시간 값. **수정 시 즉시 저장**(saveStock + 토스트), S3·재고 탭에 바로 반영.
  - 초기 칸 = 기존 그대로 (재고 초기화 기준, 저장 버튼으로 커밋).
- 번들: 부모 행 잔여 = 하위 합계(읽기 전용). **하위 행마다 잔여 칸** 추가(즉시 저장).
- 클램프: 잔여는 **0 ~ 초기값**. 초과 입력 시 초기값으로 잘리고 토스트 안내.
  실물 재고 추가 시엔 초기값 먼저 올리고 → 잔여 올리는 순서.
- 재고 탭은 읽기 전용 현황판 유지.

## 기능 2 — 상위 등급 재당첨 방지 쿨다운

- **범위**: `tier:'high'` 상품이 당첨되면 **당첨된 그 등급만** N분간 추첨 풀에서 제외. (IP별 독립)
- **설정**: 관리자 설정 탭 "상위 쿨다운(분)" 입력, `cfg.cooldownMin`. 기본 15, **0 = 기능 끔**(긴급 해제 겸용). 기존 설치엔 필드 부재 → 15로 간주.
- **저장**: 새 localStorage 키 `kuji.cooldown.v1` — `"ipId|등급명" → 만료시각(epoch ms)`.
  - 새로고침·재시작에도 유지. 등급명 키라 순서 변경(▲▼)에 안전.
  - `.kuji` 백업에 **미포함** (쿨다운은 기기 간 이전 대상 아님).
- **추첨** (`pickDraw`): 풀 구성 시 쿨다운 중 상위 등급 제외 → **필터 후 풀이 비면 쿨다운 무시하고 전체 풀** (100% 당첨 보장). 상위 당첨 시 쿨다운 등록.
- **표시**: 참여자 화면(S3)에는 숨김. 관리자 **재고 탭**에만 `🔒 n분` 표시.

## 변경 없음

행운상 판정, 번들 추첨(재고 비례), 로그/CSV, 일련번호, 화면 플로우.

## 검증

- Node 문법 체크.
- 추첨 시뮬레이션: (1) 전량 정확 소진(100% 당첨 유지) (2) 쿨다운 중 상위 미당첨 (3) 쿨다운 상품만 남은 경우 폴백 지급 (4) 행운상 정상 발동.
- `APP_VER v19→v20`, `sw.js CACHE kuji-v20`.
