# 플랜 — 컴투스 스토어 디자인 시스템을 쿠지 앱에 적용

## Context (왜 이 작업을 하는가)
쿠지 이벤트 앱(`kuji-app/index.html`)은 1차 구현이 끝났고 기능은 동작한다. 다만 현재 외관은 프로토타입에서 이어받은 **다크 보라 + 옐로** 톤이라, 사내 **컴투스 스토어(https://com2ustore.com)** 의 디자인 시스템과 통일이 필요하다.

진행 합의:
- Jason이 **내일** 파트너사/팀원에게서 스토어 **피그마 시안**을 받아 → Claude로 **디자인 시스템화** → **MD로 정제**해서 전달.
- 토큰 레이어 구축 + 실제 적용은 **MD 도착 후 한 번에** 진행(헛작업 방지).
- 테마 방향(라이트/다크·색)은 **MD가 정의하는 대로** 따른다.
- 원칙: "구조는 그대로, 비주얼만 교체." 화면 플로우·로직·관리자 기능은 건드리지 않는다.

> 이 플랜은 **MD 도착 시 실행**할 작업 정의서다. 오늘은 코드 변경 없음.

---

## 전달받을 MD 스펙 (Jason 준비물 — 이 항목들이 있으면 그대로 매핑됨)
MD에 아래 토큰이 값으로 정리되어 있으면 변수에 바로 꽂는다. (없으면 시안에서 추출)

1. **테마 베이스**: 라이트/다크 여부, 페이지 배경색
2. **컬러** (hex): primary, secondary, accent/point, 배경, surface(카드), 보더/구분선, text(기본·약), success, danger
3. **폰트**: family(예: Pretendard), 굵기 단계(제목/본문), 사이즈 스케일(선택)
4. **버튼**: 라운드(px/999), 채움(솔리드/그라데이션)·색, 그림자, hover/active
5. **카드/모서리**: 카드 배경·라운드·그림자, 보더 스타일
6. **로고/워드마크**: 이미지 or 텍스트 스타일 (대기화면 로고에 사용)
7. **무드 레퍼런스**: 1~2줄 (예: 플레이풀+프리미엄, 게이밍 톤)

## 구현 단계 (MD 도착 후)

### 1단계 — 디자인 토큰 레이어 구축 (`:root` CSS 변수)
`kuji-app/index.html` 의 `<style>` 최상단에 `:root { --token: value }` 블록을 만들고, 현재 흩어진 하드코딩 색/폰트/라운드를 변수로 치환한다. 현재 팔레트 그룹(인벤토리):

| 토큰 그룹 | 현재 값(치환 대상) |
|---|---|
| 배경(다크) | `#0e0820` `#1a1030` (admin `#15102b` `#221a45`) |
| 보라 그라데이션 | `#3b2a6e` `#5d3fa8` `#7b5bd6` (`#4527a0` `#6a4fc9` `#4a338f` `#b9a6ff`) |
| 액센트(버튼) | `#f3ef4e` `#d8d520` / 그림자 `#a8a210` / 글자 `#3a3a00` |
| 골드 | `#ffe14d` `#ffd34d` `#ffe9a8` `#ffce3d` `#b8860b` |
| 포인트(핑크) | `#ff5d8f` |
| 위험(danger) | `#d32f2f` |
| surface(카드) | `#fff` `#f1ecff` `#efe7ff/#d9c9ff` |
| 텍스트 | `#222` `#666` `#888` `#aaa`/`#bbb`/`#eee`/`#fff` |
| 뱃지 | bg `#1c1c1c` / fg `#ffe14d` |
| 라운드 | 버튼 `999px`, 카드 `14~26px` |
| 폰트 | `Pretendard, Apple SD Gothic Neo, Malgun Gothic` |

→ 변수 예: `--bg`, `--surface`, `--surface-2`, `--accent`, `--accent-grad`, `--accent-shadow`, `--gold`, `--point`, `--danger`, `--text`, `--text-dim`, `--badge-bg`, `--badge-fg`, `--radius-btn`, `--radius-card`, `--font`.

### 2단계 — MD 값 매핑
MD의 토큰을 1단계 변수에 대입. **테마가 라이트로 바뀌면** 추가로 손볼 곳:
- 대기화면(S1) 배경/영상 오버레이·로고 가독성, 버블 파티클 색
- 화면 전체 배경 그라데이션(`#kiosk` 배경), topbar 대비
- 결과/연출 카드의 텍스트 대비(라이트 배경 위 텍스트색)
- 관리자 화면(다크 고정이던 부분) 톤 정합
- 콘페티/번개 이펙트 색 (브랜드 컬러로 조정 가능)

### 3단계 — 컴포넌트 디테일 정렬
MD의 버튼/카드/타이포 규칙에 맞춰 정렬:
- 버튼 라운드·그림자, 카드 보더·그림자, 폰트 굵기/사이즈
- IP 카드(S2) · 상품 가로형 카드(S3) · 뽑기 박스+NPC 영역(S4) · 결과 카드(S6)
- 대기화면 로고(다이아) 교체 여부, 관리자 패널(현재 다크 고정) 톤 정합
- 연출(콘페티·금빛 번개) 색을 브랜드 컬러로 조정 여부

### 4단계 — 태블릿 반응형 유지
기존 `@media (min-width:600px)` 비례 확대 규칙이 새 토큰과 충돌 없게 점검(값은 변수 참조로).

## 검증
1. 로컬 서버(`python -m http.server`)로 `Ctrl+Shift+R` 후 전 화면(S1~S6 + 관리자) 색·폰트·버튼·카드가 MD대로 적용됐는지 육안 확인.
2. iPad Pro 기기 툴바로 태블릿 비율에서 레이아웃·대비 점검(라이트 전환 시 텍스트 가독성 집중 확인).
3. 기능 회귀 없는지: 뽑기→연출→결과, 행운상, 관리자 업로드, BGM/효과음, 30초 유휴 — 동작 그대로인지 클릭 테스트.
4. JS 무변경 확인(문법 체크), 추첨 로직은 손대지 않으므로 재검증 불필요.
