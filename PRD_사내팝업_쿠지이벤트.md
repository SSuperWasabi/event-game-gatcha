## 1. 개요

### 1.1 배경 및 목적
- Com2uS 사내 팝업 스토어에서 **임직원 대상 무료 이벤트 게임**으로 운영한다.
- 일본 이치방쿠지 디지털 키오스크의 UX를 레퍼런스로, **"정해진 수량에서 뽑는 복권식 가챠(쿠지)"** 경험을 제공한다.
- 자사 IP 굿즈를 경품으로 활용하여 임직원의 IP 친밀도와 이벤트 만족도를 높인다.

### 1.2 레퍼런스 분석 요약 (영상 기반)
실기 키오스크의 플로우: ① 대기 화면("화면을 터치하면 시작합니다") → ② 상품 시리즈 목록(가격·잔여량 29/80 표시) → ③ 등급별 경품 상세(A상~J상, 등급별 잔여 수량, LAST ONE상, DOUBLE CHANCE상) → ④ 참여 횟수 선택(+/−, 합계 금액) → ⑤ 카드 결제.

본 프로젝트는 이 중 **결제 단계를 제거**하고(사내 무료 이벤트), 영상에 없던 **뽑기 연출·결과 화면**을 추가한다.

### 1.3 1차 프로토타입 범위
- **포함**: 단일 이벤트 풀(시리즈 1개), 전체 키오스크 플로우, 비복원 추첨, LAST ONE상, 관리자 화면, PWA 오프라인 동작
- **제외**: 다중 시리즈, 결제, 사용자 인증/계정, 서버 연동, DOUBLE CHANCE상

## 2. 타겟 & 운영 시나리오

### 2.1 운영 환경
- **기기**: 태블릿 1대 (iPad 또는 갤럭시탭, 세로 거치)
- **장소**: 사내 팝업 스토어 내 키오스크 거치대
- **운영**: 스태프 1인 이상 상주 (참여 통제 + 굿즈 지급)

### 2.2 현장 동선 (해피 패스)
1. 임직원이 줄을 서고, 스태프가 참여 자격 확인 (1인 1회 — 명단 체크 등 현장 통제)
2. 참여자가 키오스크 앞에 서면 대기 화면 터치 → 게임 시작
3. 경품 라인업 확인 → "뽑기 시작" → 티켓 선택 → 당첨 연출
4. **결과 화면을 스태프에게 제시** → 스태프가 해당 등급 굿즈 지급
5. 스태프가 "처음으로" 버튼을 눌러 다음 참여자를 위해 대기 화면으로 복귀

### 2.3 엣지 시나리오
| 상황 | 처리 |
|---|---|
| 참여자가 결과 화면에서 이탈 (방치) | 90초 무입력 시 자동으로 대기 화면 복귀 (당첨 로그는 이미 저장됨) |
| 전체 티켓 소진 | 대기 화면·시작 버튼에 "SOLD OUT" 표시, 게임 진입 차단 |
| 태블릿 재시작/브라우저 종료 | localStorage에 재고·로그가 저장되어 있어 이어서 운영 가능 |
| 행사장 와이파이 끊김 | PWA 오프라인 캐싱으로 정상 동작 (최초 1회 로딩만 온라인 필요) |
| 시스템 재고와 실물 재고 불일치 | 관리자 화면에서 당첨 로그 CSV로 대사(對査) |

## 3. 핵심 게임 메커니즘 — 비복원 추첨 (쿠지 방식)

> 일반 확률 가챠와 다름에 유의. **물리적 복권 상자를 시뮬레이션**한다.

### 3.1 규칙
1. 이벤트 시작 시 전체 티켓 풀 = 등급별 수량의 합 (예: A상 2 + B상 3 + C상 5 + D상 10 + E상 20 = 총 40장)
2. 참여자가 1장을 뽑으면 **잔여 티켓 중에서 균등 확률로 추첨**하고, 뽑힌 티켓은 풀에서 **영구 제거**된다 (비복원).
3. 따라서 특정 등급이 소진되면 그 등급은 더 이상 나오지 않으며, 후반으로 갈수록 남은 등급의 체감 확률이 변한다.
4. **마지막 1장**을 뽑은 참여자에게는 해당 티켓의 경품 + **LAST ONE상**을 추가 증정한다.
5. 풀이 비면 SOLD OUT — 관리자가 리셋하기 전까지 게임 진입 불가.

### 3.2 구현 명세
- 추첨 로직: 잔여 수량 배열로 가중 풀을 만들고 `Math.random()`으로 1장 선택 → 해당 등급 수량 1 차감 → 즉시 localStorage 저장
- 차감 시점은 **티켓 터치 직후** (연출 시작 전). 연출 중 새로고침해도 재고는 이미 차감된 상태여야 함 (중복 지급 방지가 미지급보다 우선)
- 당첨 로그: `{ timestamp(ISO), grade, prizeName, isLastOne }` 을 배열로 localStorage에 누적

## 4. 화면 플로우 & 명세

```
[S1 대기] ─터치→ [S2 경품 라인업] ─"뽑기 시작"→ [S3 뽑기(티켓 선택)]
                                                      │ 티켓 터치
[S1 대기] ←─"처음으로"/90초─ [S5 결과] ←─터치─ [S4 당첨 연출]
[S9 관리자] ← 대기 화면 좌상단 5회 연속 터치 (숨김 진입)
```

### S1. 대기 화면 (Attract)
| 요소 | 명세 |
|---|---|
| 이벤트 로고 | 다이아몬드형 로고 (이치방쿠지 오마주), config의 `eventName` 표시 |
| 터치 유도 배너 | "화면을 터치하면 시작합니다." — 펄스 애니메이션 |
| 배경 | 보라 그라데이션 + 떠오르는 버블 파티클 (베이스 코드 재사용) |
| 잔여량 표시 | 화면 하단에 "잔여 OO / OO" 작게 표시 |
| SOLD OUT 상태 | 풀 소진 시 배너를 "금일 이벤트가 종료되었습니다"로 교체, 터치해도 진입 불가 |
| 동작 | 화면 아무 곳 터치 → S2. **좌상단 모서리 5회 연속 터치(3초 내) → S9 관리자** |

### S2. 경품 라인업
| 요소 | 명세 |
|---|---|
| 상단 바 | 이벤트명 + "잔여량 N/M" + 뒤로가기(→S1) |
| 등급 카드 그리드 | 2열 그리드. 각 카드: 등급 뱃지(A상~), 경품 이미지(`config.prizes[].image`), 경품명, "잔여 n / 초기수량", 잔여 게이지 바. 소진 등급은 회색 처리 + "품절" |
| LAST ONE상 카드 | 그리드 하단 전체폭, 금색 배경, "마지막 티켓을 뽑으면 추가 증정!" |
| 하단 고정 버튼 | "뽑기 시작" (노란색 대형 버튼) → S3. 풀 소진 시 비활성 |

### S3. 뽑기 — 티켓 선택
| 요소 | 명세 |
|---|---|
| 안내 문구 | "마음에 드는 티켓을 터치하세요!" |
| 티켓 그리드 | 접힌 제비 티켓 12개(잔여가 12 미만이면 잔여 수만큼) 표시, 흔들림 애니메이션. **어느 티켓을 고르든 결과는 동일한 추첨 1회** (선택은 연출일 뿐) |
| 동작 | 티켓 터치 → 즉시 추첨·차감·로그 저장 → 티켓 확대/개봉 연출 → S4 |
| 이탈 방지 | 이 화면 진입 후에는 뒤로가기 없음 (1인 1회 보장) |

### S4. 당첨 연출
| 요소 | 명세 |
|---|---|
| 당첨 카드 | 등급 뱃지 + 경품 이미지 + 경품명. 스프링 팝 등장 애니메이션 |
| 상위 등급 연출 | `config.prizes[].tier === "high"` 인 등급: 금색 카드 배경 + 콘페티 + (P1) 팡파레 사운드. 연출 시작 전 0.8초 "두근두근" 지연 |
| LAST ONE | 마지막 티켓인 경우, 당첨 카드 확인 후 LAST ONE상 카드를 한 번 더 연출 (연속 2장) |
| 동작 | 화면 터치 → S5 |

### S5. 결과 화면 (스태프 제시용)
| 요소 | 명세 |
|---|---|
| 당첨 내역 카드 | 등급·경품명·이미지 크게 표시 (LAST ONE 포함 시 2줄). **스태프가 한눈에 식별 가능해야 함** — 등급 뱃지를 크고 명확하게 |
| 발급 정보 | 당첨 시각(HH:MM:SS) + 일련번호(당일 누적 회차) — 스태프 검증용 |
| 안내 문구 | "이 화면을 스태프에게 보여주세요!" |
| 동작 | "처음으로" 버튼 → S1. 90초 무입력 → 자동 S1 |

### S9. 관리자 화면 (숨김)
| 요소 | 명세 |
|---|---|
| 진입 | S1 좌상단 5회 연속 터치 → PIN 입력 (config의 `adminPin`, 기본 `0000`) |
| 재고 현황 | 등급별 잔여/초기 수량 테이블 + 전체 잔여량 |
| 당첨 로그 | 최근 당첨 내역 리스트 (시각·등급·경품명) |
| CSV 내보내기 | 당첨 로그 전체를 CSV 파일로 다운로드 (`kuji-log-YYYYMMDD.csv`) |
| 재고 리셋 | 2단계 확인(버튼 → "정말 초기화?" 컨펌) 후 재고·로그 전체 초기화 |
| 닫기 | "운영 화면으로" 버튼 → S1 |

## 8. Opus 구현 지시사항

### 8.1 베이스 코드
`Event Game_Gatcha/kuji-event-game.html` 을 **참고 베이스**로 사용하라. 화면 디자인 톤(보라 그라데이션, 노란 버튼, 다이아 로고, 티켓 연출)과 비복원 추첨 로직은 검증되었으니 재사용하되, 본 PRD와 다른 부분(다중 시리즈→단일 풀, 결제 화면 제거, 참여횟수 선택 제거→1인 1회, config 외부화, PWA화, 관리자 화면 신설)은 본 PRD를 따른다.

### 8.2 파일 구조
```
kuji-app/
├── index.html          # 앱 본체 (화면·로직, CSS/JS 인라인 허용)
├── config.json         # 이벤트 설정 (아래 스키마)
├── manifest.webmanifest # PWA 매니페스트 (세로 고정, fullscreen)
├── sw.js               # service worker (전체 자산 사전 캐싱, 캐시 버전 관리)
├── assets/
│   ├── prizes/         # 등급별 경품 이미지
│   ├── key-visual.png  # (선택) 대기 화면 비주얼
```

### 8.3 config.json 스키마 (P2 다중 시리즈 대비, 배열로 설계)
```json
{
  "eventName": "컴투스 팝업 쿠지",
  "adminPin": "0000",
  "idleTimeoutSec": 90,
  "series": [
    {
      "id": "main",
      "name": "서머너즈 워 컬렉션",
      "lastOne": { "name": "스페셜 컬러 피규어", "image": "assets/prizes/lastone.png" },
      "prizes": [
        { "grade": "A상", "name": "발키리 피규어",      "image": "assets/prizes/a.png", "count": 2,  "tier": "high"   },
        { "grade": "B상", "name": "이프리트 피규어",    "image": "assets/prizes/b.png", "count": 3,  "tier": "high"   },
        { "grade": "C상", "name": "빅 쿠션",           "image": "assets/prizes/c.png", "count": 5,  "tier": "normal" },
        { "grade": "D상", "name": "아크릴 스탠드(랜덤)", "image": "assets/prizes/d.png", "count": 10, "tier": "normal" },
        { "grade": "E상", "name": "스티커 세트",        "image": "assets/prizes/e.png", "count": 20, "tier": "normal" }
      ]
    }
  ]
}
```
- 1차 프로토타입은 `series[0]`만 사용한다.
- `image`가 없거나 로드 실패 시 placeholder(이모지 또는 등급 텍스트)로 폴백 — 이미지 미준비 상태에서도 전체 플로우가 동작해야 함.
- config 변경 감지: localStorage에 config 해시(또는 수량 합계+이벤트명)를 저장해두고, 불일치 시 관리자에게 "설정이 변경되었습니다. 재고를 초기화할까요?" 안내.

### 8.4 배포 & 운영 절차 (README에 포함할 것)
1. 정적 호스팅(GitHub Pages 또는 Vercel)에 `kuji-app/` 업로드 (HTTPS 필수 — service worker 요건)
2. 태블릿 브라우저로 URL 접속 → 전체 로딩 확인 → "홈 화면에 추가"
3. 홈 화면 아이콘으로 실행 → 관리자 화면에서 재고 확인 → 운영 시작
4. 콘텐츠 수정 시: config.json/이미지 교체 업로드 → 태블릿에서 온라인 상태로 새로고침 (sw 캐시 버전 갱신)
5. 키오스크 잠금: iPad는 설정 > 손쉬운 사용 > 사용법 유도(Guided Access), 갤럭시탭은 앱 고정 기능 사용
